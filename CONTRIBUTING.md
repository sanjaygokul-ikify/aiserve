# Contributing to aiserve
## Guidelines
* Fork the repository and create a new branch for your changes
* Ensure all changes are formatted according to PEP 8 standards
* Include a clear and descriptive commit message
* Submit a pull request with a detailed description of your changes

## Commit Message Format
* Use the imperative mood (e.g., 'Add feature' instead of 'Added feature')
* Keep the first line concise (<50 characters) and focused on the change
* Use a blank line to separate the brief summary from the body

## Code Standards
* Follow PEP 8 conventions for coding style and formatting
